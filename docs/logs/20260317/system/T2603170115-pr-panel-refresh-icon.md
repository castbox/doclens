# 任务简报：PR 文件面板增加刷新按钮

## 1. 变更摘要

- 在右侧 PR 文件面板头部新增刷新 icon 按钮，用于手动拉取最新 PR 文件快照。
- 刷新按钮复用现有静默刷新体验：点击后不清空当前列表，仅展示“正在后台刷新...”状态。
- `GET /api/reviews/pr-files` 新增可选查询参数 `refresh=1`，命中后会先执行一次 PR 快照同步，再返回最新列表数据。

## 2. 行为保持说明

- 保持原有自动刷新、筛选、星标、已读状态、打开文件等交互不变。
- 未新增新的接口路径，仅在既有 PR 文件列表接口上增加可选强制同步参数，避免改动调用面。
- 手动刷新后仍然沿用现有 PR 快照与 SQLite 状态合并逻辑，不改变已读/星标数据来源。

## 3. 影响文件

- `app/api/reviews/pr-files/route.ts`
- `src/features/reviews/ui/ReviewDrawer.tsx`

## 4. 验证记录

- 执行 `pnpm build`
- 结果：通过

## 5. 风险与后续

- 当前“手动刷新”会触发一次服务端 PR 快照扫描；在 PR 文件规模继续增长时，可再评估是否需要节流提示或返回最近同步时间。
