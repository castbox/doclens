# 文档详情页加载提速简报

- 时间：2026-02-07 03:50
- 目标：提高 `/docs` 文档详情页加载速度与交互响应速度。

## 1. 优化项

### 1.1 目录树改为真正懒加载
- 文件：`src/features/docs/ui/DocsTree.tsx`
- 原行为：页面初始化时递归请求并展开全量目录（目录多时首屏慢）。
- 新行为：
  1. 首屏仅加载根目录
  2. 点击目录时按需加载子目录
  3. 从搜索或直链进入深层文件时，仅加载该文件祖先路径链路
  4. 增加请求去重（inflight map），避免并发重复请求同一路径
- 预期收益：显著减少首屏 API 请求数量与渲染节点数量。

### 1.2 文档预览增加内存缓存
- 文件：`src/features/docs/ui/DocPreview.tsx`
- 新行为：
  1. 最近预览文件缓存（LRU 风格，最多 80 项）
  2. 二次打开同一文件直接命中缓存，跳过网络请求
- 预期收益：文件来回切换时详情区秒开，减少 API 请求。

## 2. 行为保持说明

- 未修改 docs 路径安全、文件读取、搜索 API、审阅流转逻辑。
- 仅调整前端加载策略与缓存策略，外部接口不变。

## 3. 验证

- 执行：`pnpm build`
- 结果：通过（编译、类型检查、静态生成均成功）

## 4. 风险与后续

1. 目录筛选在懒加载模式下只对已加载节点生效（符合按需加载策略）。
2. 预览缓存为内存级；如需跨刷新保持可考虑后续引入 sessionStorage。
