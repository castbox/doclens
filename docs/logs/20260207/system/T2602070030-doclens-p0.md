# DocLens P0 实施简报

## 1. 变更目标

依据 `docs/prd/doclens_prd.md` 实现 P0 范围：

- docs 目录树浏览与文件预览
- rg 全文搜索（内联 + 独立页）
- 审阅单创建、列表、详情、问题项、状态流转
- 路径安全与 docs 只读约束
- 最小回归测试与构建校验

## 2. 核心实现

### 2.1 工程与架构

- 初始化 Next.js App Router + TypeScript + MUI 工程
- 采用 feature 分层：
  - `src/features/docs/{ui,domain,services}`
  - `src/features/search/{ui,domain,services}`
  - `src/features/reviews/{ui,domain,services}`
  - `src/shared/{domain,ui,utils}`

### 2.2 Docs 浏览

- `GET /api/docs/tree?path=...`：懒加载目录树
- `GET /api/docs/file?path=...`：文本/markdown/code 预览
- `GET /api/docs/file?path=...&raw=1`：原文件流式读取（pdf/binary）
- `GET /api/docs/meta?path=...`：文件元信息
- `/docs` 页面实现：左树 + 中预览 + 右审阅抽屉

### 2.3 搜索

- `GET /api/search` 使用 `rg --json` 检索
- 支持关键词、分页、类型过滤、排序
- 关键词高亮与命中跳转（`#Lxx`）
- 双入口：
  - `/docs` 内联搜索面板
  - `/search` 独立搜索页

### 2.4 审阅单

- SQLite + Drizzle 数据模型：`review_sheets` / `review_items`
- 状态机：`Draft -> In Review -> Changes Requested -> Approved -> Done`
- API：
  - `POST/GET /api/reviews`
  - `GET/PATCH /api/reviews/:id`
  - `POST /api/reviews/:id/items`
  - `PATCH /api/reviews/:id/items/:itemId`
- UI：右侧抽屉创建与关联列表、`/reviews` 列表、`/reviews/:id` 详情与问题项编辑

### 2.5 安全与只读

- `resolve + normalize + root check`
- 拒绝绝对路径、`..` 穿越
- docs API 仅提供读取，不提供写入
- 审阅数据仅写入 SQLite
