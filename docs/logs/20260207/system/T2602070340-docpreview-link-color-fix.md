# DocPreview 文档渲染修复简报

- 时间：2026-02-07 03:40
- 目标：修复文档渲染时自动链接与代码样式冲突问题，并将代码色调整为目标橙色风格。

## 1. 问题现象

在 Markdown 渲染中，部分已是链接语法或代码片段的内容会被自动补链逻辑再次处理，导致显示出原始 `[...](...)` 文本。并且代码相关配色与目标样式不一致。

## 2. 修改内容

- 文件：`src/features/docs/ui/DocPreview.tsx`
- 关键改动：
  1. 自动补链逻辑增加“受保护区间”判断：
     - 跳过行内代码（`` `...` ``）
     - 跳过已有 Markdown 链接（`[...](...)`）
  2. 文档内代码颜色改为橙色系：
     - 代码文本颜色：`#FF6A00`
     - 行内代码背景：`rgba(255,106,0,0.12)`
     - 代码块背景：`#FFF8F1`
  3. 代码行预览（code/text）文本颜色同步为橙色。

## 3. 行为影响

- 仅影响 `/docs` 预览页渲染层。
- 不涉及 API、搜索逻辑、数据库模型。
- 链接识别策略更保守，避免对已有语法进行二次改写。

## 4. 验证

- 构建命令：`pnpm build`
- 结果：通过（编译、类型检查、静态页面生成均成功）。

## 5. 风险

- 当前“受保护区间”规则基于单行正则（行内代码与链接），可覆盖当前问题；如后续出现更复杂嵌套 Markdown 语法，建议改为 remark AST 级处理以获得更强鲁棒性。
